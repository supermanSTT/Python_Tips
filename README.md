# Python_Tips

Python 환경을 관리할 때 가장 많이 언급되는 두 도구, **pyenv**와 **uv**를 비교해 드립니다.

과거에는 `pyenv`가 정석이었지만, 최근에는 Rust 기반의 초고속 도구인 `uv`가 생태계를 뒤흔들고 있습니다. 한마디로 요약하자면 **"전통의 강자 vs 차세대 올인원 플레이어"**의 대결입니다.

---

### 📊 pyenv vs uv 한눈에 비교

| 구분 | **pyenv** | **uv** |
| --- | --- | --- |
| **핵심 목적** | 여러 버전의 Python 설치 및 관리 | Python 버전 + 패키지 + 가상환경 관리 |
| **주요 기술** | Shell Scripts (Bash 기반) | **Rust** (압도적인 속도) |
| **패키지 관리** | 지원 안 함 (pip 별도 사용) | **직접 지원** (pip보다 수십 배 빠름) |
| **설치 방식** | Shell 설정 및 빌드 환경 필요 | 단일 바이너리 (설치가 매우 간편) |
| **의존성 해결** | 없음 | 고도로 최적화된 Resolver 탑재 |
| **철학** | "Python 버전만 확실히 바꿔줄게" | "Python 개발에 필요한 모든 걸 해결해줄게" |

---

### 1. pyenv: "버전 관리의 교과서"

Python 버전 자체를 시스템 전역 혹은 프로젝트별로 스위칭하는 데 최적화되어 있습니다.

* **장점:** * 오랫동안 검증된 안정성.
* 거의 모든 Python 버전(PyPy, Anaconda 등 포함)을 지원.
* `.python-version` 파일을 통해 팀원들과 버전을 맞추기 쉬움.


* **단점:** * Python을 소스에서 직접 빌드하므로 설치 시간이 길고, 시스템 라이브러리 의존성이 까다로움.
* 가상환경 관리를 위해 `pyenv-virtualenv` 같은 플러그인을 추가로 써야 함.



### 2. uv: "속도에 미친 올인원 툴"

Astral(Ruff 개발사)에서 만든 도구로, 현재 Python 생태계에서 가장 핫한 도구입니다.

* **장점:** * **압도적 속도:** 패키지 설치나 버전 다운로드가 눈 깜짝할 새 끝납니다.
* **올인원:** `pyenv` + `pip` + `venv` + `pip-tools` 기능을 하나로 합쳤습니다.
* **환경 보존:** 시스템 Python에 영향을 주지 않고 프로젝트마다 독립적인 Python 실행 파일을 순식간에 내려받아 사용합니다.


* **단점:** * 비교적 신생 도구라 아주 특수한 레거시 환경에서의 호환성은 검증이 더 필요할 수 있음.
* `pyenv`만큼 다양한 변종 Python(특수 빌드 등) 지원은 아직 부족할 수 있음.



---

### 💡 결론: 어떤 것을 써야 할까요?

* **이런 분께 `pyenv`를 추천합니다:**
* "나는 도구들이 각각 자기 역할만 하는 게 좋아(Unix 철학)."
* 특이한 버전의 Python이나 매우 안정적인 환경이 최우선이다.
* 이미 기존 워크플로우가 `pyenv`에 맞춰져 있어 바꿀 이유가 없다.


* **이런 분께 `uv`를 강력 추천합니다:**
* **"속도가 제일 중요하다."** (pip 설치 기다리는 시간이 아깝다.)
* 버전 관리부터 패키지 설치까지 도구 하나로 끝내고 싶다.
* Docker 빌드 시간을 획기적으로 줄이고 싶다.
